🖥️EoSim — Official Simulation Platform

EoSim is the multi-architecture embedded simulation platform — supporting 52+ platforms across 12 architectures with native Python simulation, QEMU, Renode, VirtualBox, VMware, Docker-like layers, and snapshots.

🖥️ EoSim Engine

52+ platform definitions — ARM, RISC-V, Xtensa, x86, MIPS, PowerPC. Native + QEMU + Renode + VirtualBox + VMware engines.

Python · 12 Architectures

🎨 14 3D Renderers

Aircraft, Vehicle, Drone, Robot, Medical, Satellite, Gaming, Weather, Finance, Aerodynamics, Physiology, Consumer, Generic — all reusable.

Ported to JS for browser

📊 GUI Dashboard

GPIO panel, UART terminal, CPU state, memory view, peripheral registers, 3D product viewer — Tkinter + Web.

7 Widget Panels

📦 VirtualBox + VMware

Generate .vbox + .vdi for VirtualBox and .vmx + .vmdk for VMware directly from platform hardware configs.

eosim vbox create · eosim vmware create

🐳 Docker-like Layers

Incremental layered VM builds — base (FROM), overlay (COPY/ADD), commit (tag), flatten (export), history — just like Docker.

eosim layer create/add/commit

📸 Snapshots

Save, restore, export, import VM snapshots across VirtualBox and VMware — with snapshot trees and portable .tar.gz archives.

eosim snapshot save/restore
Quick Start — EoSim
pip install git+https://github.com/embeddedos-org/eosim.git

eosim list                            # 52+ platforms
eosim run stm32f4 --gui               # launch simulation with GUI
eosim vbox create x86_64-linux        # generate VirtualBox VM
eosim vmware create raspi4            # generate VMware VM
eosim snapshot save my-vm checkpoint1 # save snapshot
eosim layer create my-vm ./out/vbox/my-vm  # create base layer
eosim layer add my-vm ./apps --name apps   # add overlay layer
eosim layer commit my-vm v1.0              # tag as v1.0

🌐eBrowser — Embedded Web Browser

eBrowser is the EmbeddedOS lightweight web browser engine — built from scratch in C/C++ for embedded devices, kiosks, and IoT dashboards. It renders the EoSim Play games natively and powers the browser extension platform.

🌐 eBrowser Engine

Lightweight HTML5/CSS3 rendering engine with DOM, layout, and networking — all under 1 MB binary.

C/C++ · LVGL · SDL2 · WASM

🧩 Extension Support

eBrowser loads eOffice Suite extensions and EoSim Play sims as browser-native apps — same extensions run in Chrome, eBrowser, and mobile.

Same JS · Same HTML

🖥️ 3 Backends

SDL2 (desktop), EoS native (embedded framebuffer), WASM (run in any browser via Emscripten).

Cross-Platform
Quick Start — eBrowser
# Linux / macOS
git clone --recursive https://github.com/embeddedos-org/eBrowser.git
cd eBrowser && ./setup.sh

# Windows
git clone --recursive https://github.com/embeddedos-org/eBrowser.git
cd eBrowser && setup.bat

# Open EoSim Play games in eBrowser
./eBrowser https://embeddedos-org.github.io/eApps/

eb_destroy(browser);

🔄Code Reuse Architecture

EoSim Python Renderers (source of truth) eosim/gui/renderers/aircraft.py ─┐ eosim/gui/renderers/vehicle.py ─┼──→ eApps/shared/js/sim-engine.js (ported geometry + math) eosim/gui/renderers/drone.py ─┘ │ ├──→ Browser Extension (eosim-play.html) ├──→ Mobile PWA (mobile-apps/eosim-play/) └──→ GitHub Pages (embeddedos-org.github.io/eApps/) Layer Reuse (Docker-like): ┌────────────────────────────────────────────┐ │ Layer 3: Custom Designs (per-deployment) │ ← eosim layer add ├────────────────────────────────────────────┤ │ Layer 2: App Layer (sim-engine.js) │ ← eosim layer add ├────────────────────────────────────────────┤ │ Layer 1: Base OS (VirtualBox / VMware) │ ← eosim layer create └────────────────────────────────────────────┘

🧩Browser Extension

The EoSim sims are integrated into the eOffice Suite Chrome extension. Click any sim in the extension popup to launch a full-screen interactive game tab.

Install
  1. Clone eApps repo
  2. Open chrome://extensions
  3. Enable Developer Mode → Load Unpacked → select extensions/browser/
  4. Click the extension icon → scroll to "🎮 EoSim Simulations"

💻EoSim CLI Commands

📸 Snapshot Commands

CommandDescription
eosim snapshot save <vm> <name>Save a snapshot of a VirtualBox or VMware VM
eosim snapshot restore <vm> <id>Restore a VM to a saved snapshot state
eosim snapshot listList all saved snapshots
eosim snapshot delete <vm> <id>Delete a snapshot and its files
eosim snapshot export <vm> <id>Export snapshot to portable .tar.gz
eosim snapshot import <vm> <archive>Import snapshot from archive

🖥️ VMware Commands

CommandDescription
eosim vmware create <platform>Create VMware .vmx + .vmdk from platform config
eosim vmware info <platform>Show VMware config mapping for a platform
eosim vmware export-ova <vmx>Export to OVA format (requires ovftool)

🐳 Layer Commands (Docker-like)

CommandDescription
eosim layer create <vm> <dir>Create base layer from VM directory (like Docker FROM)
eosim layer add <vm> <dir>Add overlay layer on top (like Docker COPY/ADD)
eosim layer commit <vm> <tag>Tag current layer stack (like Docker tag/commit)
eosim layer listList all layers across all VMs
eosim layer history <vm>Show layer history (like docker history)
eosim layer flatten <vm>Merge all layers into single directory (like docker export)
eosim layer imagesList all VM images with layer counts and tags

📦 VirtualBox Commands

CommandDescription
eosim vbox create <platform>Create VirtualBox .vbox + .vdi from platform config
eosim vbox listList VirtualBox VMs (requires VBoxManage)
eosim vbox info <platform>Show VirtualBox config mapping for a platform
